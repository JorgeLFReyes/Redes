Seguridad WPA2

WPA2(Acceso protegido Wi-Fi 2) es un protocolo cifrado de seguridad que protege el tráfico de Internet en redes inalámbricas.

La segunda generación del protocolo de seguridad de Acceso protegido Wi-Fi(WPA2) aborda errores anteriores y ofrece un cifrado más potente.

Desde su lazamiento en 2004, se ha convertido en el estándar de seguridad de las redes de WiFi

Funcionamiento

WPA2 o también llamado RSN(Road Segurity Network) desarrollado por la wifi alliance surge del estandar y triple 802.11 2004 después de la versión de transición que fue el WPA, este estándar introduce dos nuevos protocolos: Four way handshake y Group key handshake

La segunda generación del protocolo de seguridad de Acceso protegido Wi-Fi(WPA2) aborda errores anteriores y ofrece un cifrado más potente.

Desde su lazamiento en 2004, se ha convertido en el estándar de seguridad de las redes de WiFi

La autenticación se divide en una versión de clave pre-compartida WPA2 personal y una con autenticación 802.11X AP, WPA2 enterprise. La autenticación inicial es llevada a cabo usando una clave compartida PSK o haciendo un intercambio mediante el EAP, después de esto una llave secreta compartida es generada llamada PMK, en donde esta será igual a PSK en caso de usarse WPA2 personal o será derivado de los parámetros EAP dados por el servidor de autenticación en mi caso de WPA2 Enterprise, IEEE 800 2.1X este estándar introduce un método seguro para poder autenticar a usuarios que quieren acceder a la red y para el intercambio de claves criptográficas de manera segura.

Se basa en el protocolo EAP sobre una red LAN llamado apple EAPol, esta forma de autenticación consta de tres partes: un suplicante, una auténticador y un servidor de autenticación.

  • El suplicante es el dispositivo cliente que quiere conectarse a la red.
  • La auténtica dor es un punto de acceso.
  • El servidor de autenticación es aquel que ejecuta software que soporta radios.

CCM

En CCM se utiliza primero el CBC-MAC se obtiene una etiqueta para la autenticación y luego tanto el mensaje como la etiqueta se cifran con el modo counter y con AES, el tamaño del bloque debe de ser de 128 bits y sirve para darle confidencialidad al mensaje.

CCMP

El protocolo plantea la utilización de información del encabezado mac, la dirección del transmisor junto con el número de paquete para crear un nons a usar durante ccm.

Utiliza una llave temporal, el nons es la información adicional para la autenticación construido a partir del encabezado mac y la información a transmitir, todo esto es autenticado con un nick que también es cifrado al mismo tiempo que se cifra se crea un encabezado ccmp que se unirá con un encabezado mac, la información cifrada y el FCS para la detección y corrección de errores.

Fort Wayne handshake

Diseñado para que el punto de acceso y el suplicante puedan probar de manera independiente que cada uno conoce el PMK sin revelar la misma, lo que hacen es inflar mensajes para que sean descifrados por su contraparte con el PMK por lo que la correcta obtención del texto plano indicará el conocimiento de la llave.

El PMK dura por toda una sesión por lo que para evitar su exposición se derivan otras claves, se establece entonces una llave a partir de la concatenación del PMK einons, nons desde el punto de acceso es nons del suplicante y la dirección mac del punto de acceso y el suplicante.

Después se utiliza una función pseudo aleatoria con esta concatenación para generar por fin la PTK de este proceso también se generará la group de PTK que se utilizará para descifrar el tráfico multicasting Broadcast.

PTK

Intercambio de mensajes:

  1. El punto de acceso manda el ANonce al cliente.
  2. El suplicante manda su SNonce al punto de acceso junto con su MAIC (Message Authentication and Integrity Code).
  3. El punto de acceso manda la GTK y un numero de secuencia junto con otro MIC. Este numero será utilizado con los paquetes de multicast y broadcast.
  4. El suplicante manda una confirmación al punto de acceso.

Group key handshake

La GTK usada en una red deberá ser actualizada al expirar un tiempo predefinido, así como cuando un dispositivo de la misma. Esta llave también contendra una llave temporal usada para cifrar y descifrar los paquetes asociados de 16 bytes.

Pasos siguientes:

  1. El punto de acceso manda la nueva GTK a cada suplicante en la red donde está se cifra con el que está asignado al suplicante y se utiliza un mic.
  2. El suplicante confirma la nueva GTK y hace una réplica al punto de acceso.
